# Roadmap

This is a personal project. Items here represent things that would make the fleet more useful or easier to adopt — not a committed delivery schedule.

---

## Near-term (next 1–3 months)

### `setup.sh` installer
Currently the biggest friction point. A script that:
- Prompts for Discord bot token and AI API key
- Creates the `openclaw.json` config
- Copies all agent SOUL.md / IDENTITY.md files into the correct workspace directories
- Registers all cron jobs
- Starts the gateway

### Complete Zalo bridge first-run UX
- Automate the QR scan step with a terminal display (no external image viewer needed)
- Add session health check so the bridge auto-reconnects on stale sessions

### Skill credential setup script
- Interactive script to walk through setting Gmail App Password, Notion token, etc.
- Validates each credential before writing it to the config

### Linux / Docker support
- Test the full setup on Ubuntu
- Provide a `docker-compose.yml` as an alternative to local install
- Document any macOS-specific dependencies that need replacements

---

## Medium-term (3–6 months)

### More agent examples
- Email triage and draft agent (uses imap-smtp-email skill)
- Notion daily sync agent
- GitHub PR summary agent
- LinkedIn research agent

### Fleet health dashboard
- A simple HTML page (served locally) showing agent status, last run times, error counts
- Could be generated by the `healthcheck` agent and posted to Discord

### Multi-model routing
- Route different agents to different models based on cost/capability tradeoffs
- Cheap model for summarize, stronger model for code/analysis agents

### Automated agent improvement loop validation
- Actually measure whether `self-improving` suggestions result in measurably better agent output
- Currently the loop exists but there is no feedback metric

---

## Long-term / stretch goals

### Agent marketplace format
- Standardize `SOUL.md` schema so agents can be shared and installed like packages
- `openclaw agent install <url>` that pulls an agent config and registers it

### Multi-Discord-server support
- Run the same fleet across multiple Discord servers (e.g. personal + work)

### Web UI for non-technical users
- A simple form-based interface to configure agents without editing JSON
- Would significantly lower the barrier to adoption

---

## Not planned

- Cloud hosting / SaaS version (out of scope for a personal project)
- Mobile app
- Support for Slack, Teams, or other chat platforms (Discord-first by design)
- Paying for or building integrations with closed/paid-only APIs
