# Roadmap

This is a personal project. Items here represent things that would make the fleet more useful or easier to adopt — not a committed delivery schedule.

---

## Near-term (next 1-3 months)

### `setup.sh` installer
Currently the biggest friction point. A script that:
- Prompts for Discord bot token and AI API key
- Creates the `openclaw.json` config
- Copies all agent SOUL.md / IDENTITY.md files into the correct workspace directories
- Registers all cron jobs
- Starts the gateway

### CI and validation
- GitHub Actions: shellcheck, markdownlint, JSON validation, secret scanning
- `scripts/validate.sh` — checks all agent dirs have required files

### Complete Zalo bridge first-run UX
- Automate the QR scan step with a terminal display
- Add session health check so the bridge auto-reconnects

### Linux / Docker support
- Test the full setup on Ubuntu
- Provide a `docker-compose.yml` as an alternative to local install
- Document any macOS-specific dependencies that need replacements

---

## Medium-term (3-6 months)

### Financial intelligence improvements
- Broker API integration (Alpaca, Interactive Brokers) for live price feeds
- Signal quality tracking — record hit rate of each setup grade over time
- Backtesting framework — validate signal-scanner setups against historical data
- Learning loop — feed signal outcomes back into prompt refinement
- Multi-timeframe analysis — weekly + daily regime alignment

### More agent examples
- Email triage agent (imap-smtp-email skill)
- Notion daily sync agent
- GitHub PR summary agent

### Fleet health dashboard
- A simple HTML page showing agent status, last run times, error counts
- Could be generated by the `healthcheck` agent

### Multi-model routing
- Route different agents to different models based on cost/capability
- Cheap model for summarize, stronger model for code/analysis

### Self-improvement loop validation
- Actually measure whether `self-improving` suggestions result in better output
- Currently the loop runs but has no feedback metric — this is the top AI/systems gap

---

## Long-term / stretch goals

### Agent marketplace format
- Standardize `SOUL.md` schema so agents can be shared like packages
- `openclaw agent install <url>` that pulls and registers an agent config

### Multi-Discord-server support
- Run the same fleet across multiple servers (personal + work)

### Web UI for non-technical users
- Form-based interface to configure agents without editing JSON

---

## Not planned

- Cloud hosting / SaaS version
- Mobile app
- Slack, Teams, or other chat platforms (Discord-first)
